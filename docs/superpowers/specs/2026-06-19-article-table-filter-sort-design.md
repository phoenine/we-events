# 文章表格筛选与排序设计

## 目标

把文章页的公众号筛选收进表格列头，并为公众号、发布时间、活动抽取三列提供与字段特性匹配的筛选和排序能力。所有条件由后端执行，确保跨分页结果、总数和排序一致。

## 交互设计

### 公众号

- 移除页面头部现有公众号下拉框。
- “公众号”列使用 Ant Design Table 内置筛选菜单。
- 筛选选项支持搜索，单次选择一个公众号。
- 公众号名称只用于筛选，不提供排序；当前名称由接口动态补充，按名称进行准确的数据库排序需要额外关联查询，投入与价值不匹配。
- 现有“采集文章”操作继续使用当前公众号筛选值；未选择公众号时保持禁用。

### 发布时间

- “发布时间”列使用自定义筛选下拉框，内容为日期范围选择器以及重置、确认按钮。
- 选择日期范围后立即回到第一页，并按闭区间查询：开始日期从当天 00:00:00 起，结束日期到当天 23:59:59 止。
- 列头支持升序和降序，默认按发布时间降序。

### 活动抽取

- “活动抽取”列使用单选状态筛选菜单，包含 `pending`、`queued`、`processing`、`fallback_required`、`failed`、`not_activity`、`extracted`。
- 列头支持升序和降序。
- 状态排序使用业务顺序，而不是字符串顺序：`processing → queued → pending → fallback_required → failed → not_activity → extracted → 其他`。
- 同一状态内按发布时间降序保持稳定结果。

### 表格状态

- 任意筛选或排序变化后回到第一页并清空批量选中项。
- 分页变化保留当前筛选和排序条件。
- 筛选和排序状态进入 React Query key，避免不同条件复用错误缓存。

## API

扩展 `GET /wx/articles`：

- `wechat_account_id`: 现有公众号 ID 筛选。
- `activity_extraction_status`: 单个活动抽取状态。
- `publish_time_from`: 可选 Unix 秒时间戳，包含边界。
- `publish_time_to`: 可选 Unix 秒时间戳，包含边界。
- `sort_by`: `publish_time` 或 `activity_extraction_status`，默认 `publish_time`。
- `sort_order`: `asc` 或 `desc`，默认 `desc`。

列表查询和总数查询必须使用同一组筛选条件。API 只允许上述排序字段，不能把客户端字符串直接拼入数据库排序表达式。

## 数据库

PostgREST 只能按真实列排序。新增存储生成列 `activity_extraction_status_order`，通过不可变 `CASE` 表达式把状态映射为业务顺序，并为状态顺序与发布时间增加索引。

后端把 `sort_by=activity_extraction_status` 映射到该生成列；该内部列不作为前端契约使用。

## 前端数据流

`listArticles` 接受新增筛选与排序参数并传给 API。`ArticlesPage` 保存受控表格状态：公众号、发布日期范围、活动状态、排序字段和排序方向。

表格 `onChange` 统一处理分页、列筛选和排序。日期范围筛选下拉框只负责生成两个 Unix 秒边界，不在前端过滤 `query.data.list`。

现有乐观删除会更新 `['articles']` 前缀下所有缓存，因此新增查询 key 维度后仍然有效。

## 测试

- 后端仓储测试验证组合筛选、计数筛选和排序表达式透传。
- API 测试验证参数白名单、时间边界和状态排序映射。
- 前端纯函数测试验证 Ant Design sorter/filter 值到 API 参数的转换，包括重置和默认排序。
- 运行完整后端相关测试、前端 Node 测试及生产构建。

## 验收标准

- 公众号列可以搜索并筛选，顶部公众号下拉框不再显示。
- 发布时间列可以选择日期范围，并按最新或最早排序。
- 活动抽取列可以筛选 `not_activity` 等状态，并按业务顺序排序。
- 筛选与排序在翻页后保持生效，返回总数准确。
- 改变筛选或排序会回到第一页并清空已选文章。
- 未选择公众号时“采集文章”按钮仍保持禁用。
