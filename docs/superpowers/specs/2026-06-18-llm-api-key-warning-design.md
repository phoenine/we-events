# LLM API Key 缺失告警设计

## 目标

当活动抽取任务因缺少 `LLM_API_KEY` 失败时，文章页面使用短暂的 `message.warning` 提示用户。其他活动抽取失败仍沿用 `message.error`。

## 范围

- 仅修改前端文章页面的活动抽取任务轮询结果处理。
- 不修改后端抽取逻辑。
- 不修改抽取任务状态或文章的活动抽取状态。
- 不引入持久化告警或全局 HTTP 错误处理。

## 行为

当任务状态为 `failed`：

1. 如果任务错误文本包含 `LLM_API_KEY`，调用 `message.warning("未配置 LLM_API_KEY，无法进行活动抽取")`。
2. 否则调用现有的 `message.error(run.error || "活动抽取失败")`。

## 验证

- 缺少密钥的失败走 warning 分支。
- 普通失败仍走 error 分支。
- 执行前端现有测试、类型检查或构建命令，确认改动不引入编译错误。
